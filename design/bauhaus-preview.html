The Bauhaus argument on colour

Bauhaus colour theory (Itten, Albers) treats colour as signal, not decoration. Every hue in the UI should answer: what does this mean? The current palette is close — amber already feels like a Bauhaus primary. The question is whether the five role/position colours are each earning their place.

A — each role has its own identity; easier to scan at a glance but adds visual noise.

B — amber = "you can earn money here" (all paid roles share it); green/purple reserved purely for gender, which is the one distinction that affects team composition. Cleaner, but roles are harder to tell apart in dense lists.

These three are functional primaries: green = go, red = stop/error, amber = action/pending. They map to Itten's primary triad and should never be used decoratively.
